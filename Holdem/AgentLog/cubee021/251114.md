### 요약 내용

**메인메뉴 및 로비 시스템 아키텍처 설계 완료**

게임 전체 플로우와 UI/UX 구조를 논의하고 최종 설계를 확정했습니다. Listen Server 기반 멀티플레이 특성과 포커 게임의 라운드 반복 특성을 고려하여 **맵 통합 방식**을 채택했습니다.

**설계 결정 사항:**

1. **플레이어 정보 설정 플로우**
   - **로컬 SaveGame 방식** 채택 (온라인 계정 시스템 제외)
   - 최초 실행 시 Sign Up 창으로 이름 입력 → SaveGame 저장
   - 이후 실행 시 저장된 이름 자동 로드
   - 이름(계정 수준)과 아이템(매치 수준)을 분리하여 관리

2. **UI 플로우 구조**
   ```
   [메인메뉴]
   ├─ SaveGame 없음 → Sign Up 창 (이름 입력)
   ├─ SaveGame 있음 → 호스트/게스트 버튼 활성화
   │
   ├─ [호스트] → 세션 생성 + 로비 진입
   │   └─ 로비에서 아이템 선택 + Ready
   │
   └─ [게스트] → 세션 목록 조회
       └─ 세션 참가 + 로비 진입
           └─ 로비에서 아이템 선택 + Ready
   ```

3. **맵 구조 설계: 통합 방식 채택**
   - **결정**: MainMenu Level + Game Level (상태 전환으로 로비/게임 처리)
   - **맵 분리 방식 기각 사유**:
     - 로비와 게임이 같은 테이블, 같은 카메라
     - 라운드 반복 플레이 특성 (로딩 없는 매끄러운 전환 필요)
     - 게임 중 자유로운 입퇴장 허용
     - Listen Server 단순 구조에 적합

4. **게임 Phase 시스템 설계**
   ```cpp
   EHoldemGamePhase (세션 수준)
   ├─ Lobby      // 대기, 아이템 선택, Ready
   ├─ Playing    // 실제 게임 진행
   └─ RoundEnd   // 10초 텀, 입퇴장 허용, 아이템 재선택 가능

   EHoldemPlayPhase (게임 수준, 기존)
   └─ Waiting → PreFlop → Flop → Turn → River → Showdown
   ```

5. **라운드 반복 플로우**
   ```
   Lobby Phase (Ready 대기)
     → 모든 플레이어 Ready → Playing Phase
     → Showdown 완료 → RoundEnd Phase (10초)
     → 10초 후 자동 Lobby Phase 복귀
     → 남은 플레이어들 다시 Ready...
   ```

6. **RoundEnd Phase 상세 (10초 텀)**
   - 승자 및 획득 칩 표시
   - 다음 라운드 카운트다운
   - **자유로운 입퇴장 허용** (새 플레이어 참가 가능)
   - **아이템 재선택 가능** (전략 변경)
   - "나가기" 버튼 활성화
   - 호스트 외 플레이어 자유롭게 세션 Leave 가능

7. **입퇴장 제어 정책**
   - **Playing Phase**: 새 플레이어 입장 제한 (관전 모드 또는 거부)
   - **Lobby/RoundEnd Phase**: 자유로운 입퇴장
   - **호스트 퇴장 시**: 세션 전체 종료 (Listen Server 특성)

8. **구현 핵심 포인트**
   - GameState에 `EHoldemGamePhase` 추가 (Replicated)
   - GameMode(서버)에서 Phase 전환 로직 관리
   - RoundEnd → Lobby 자동 전환 타이머 (10초)
   - PlayerController/Widget에서 Phase별 UI 전환
   - 같은 레벨 내에서 상태만 전환 (ServerTravel 불필요)
