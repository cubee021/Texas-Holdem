#### 2. 멀티플레이 버그 수정 (PlayerArray 순서 문제)

**증상:**
```
Player 2 (Big Blind)에게 턴이 넘어갔지만 UI가 표시되지 않음
로그: [OnTurnChangedHandler] BOOK-HL5B6BLA2E-FD84 (Index 0) | NewTurnIndex: 2 | IsMyTurn: FALSE
→ Player 2가 자신의 Index를 0으로 인식!
```

**원인:**
- GameState의 PlayerArray는 각 클라이언트에서 PlayerState가 추가되는 순서대로 구성됨
- 서버와 클라이언트의 PlayerArray 순서가 다를 수 있음
- IsMyTurn() 로직이 `PlayerArray.Find(PS)`를 사용하여 클라이언트마다 다른 인덱스 반환

**해결:**

**① HoldemPlayerState.h에 SeatIndex 추가**
```cpp
// Seat 번호 (접속 순서대로 0, 1, 2, ...)
UPROPERTY(Replicated, BlueprintReadOnly, Category = "Player")
int32 SeatIndex = -1;
```
- GetLifetimeReplicatedProps()에 등록

**② HoldemGameMode::PostLogin() 수정**
```cpp
// SeatIndex 할당
if (AHoldemPlayerState* PS = NewPlayer->GetPlayerState<AHoldemPlayerState>())
{
    PS->SeatIndex = GS->PlayerArray.Find(PS);
    UE_LOG(LogTemp, Warning, TEXT("[PostLogin] %s assigned SeatIndex: %d"),
        *PS->GetPlayerName(), PS->SeatIndex);
}
```

**③ MyPlayerController::IsMyTurn() 수정**
```cpp
bool AMyPlayerController::IsMyTurn() const
{
    AHoldemGameState* GS = GetWorld()->GetGameState<AHoldemGameState>();
    AHoldemPlayerState* PS = GetPlayerState<AHoldemPlayerState>();
    if (!GS || !PS) return false;

    // ✅ SeatIndex 기반으로 판단
    return GS->CurrentTurnPlayerIndex == PS->SeatIndex;
}
```

**결과:**
✅ 3인 플레이에서 Player 2의 UI가 정상적으로 표시됨
✅ 모든 플레이어가 정확한 턴에 베팅 버튼 활성화
