### 기술적 의사결정

**1. Attach vs Physics Handle**
- 멀티플레이 동기화 효율을 고려해 Attach 방식 선택
- 부모 Transform만 Replicate하면 자식도 자동 동기화
- Physics Handle은 매 프레임 물리 상태 전송 필요

**2. Character vs Controller 책임 분리**
- Ray Trace 로직은 MyPlayer (Character의 능력)
- Input 처리는 MyPlayerController
- AI도 재사용 가능하도록 설계

**3. GameState 활용**
- PlayerArray로 접속자 수 추적
- 클라이언트도 접근 가능한 Replicated 데이터
- GameMode의 중복 데이터 관리 제거
