## 작업 내용

### 문제 상황

1. **Community Cards**: `FRotator::ZeroRotator` 사용으로 테이블 그림과 30도 어긋남
2. **Player Cards**: `Player->GetActorForwardVector()` 사용으로 플레이어가 카메라를 움직이면 카드 스폰 위치도 변경됨

### 실패한 방식 (Before)

#### Player Cards
```cpp
// ❌ 문제: 플레이어의 시선(ForwardVector)에 종속
APawn* Player = PS->GetPawn();
FRotator CardRotation = FRotationMatrix::MakeFromYZ(
    Player->GetActorForwardVector(), FVector::UpVector).Rotator();

// 카드 위치도 플레이어의 RightVector 사용
OutFirstCardLoc = BaseLocation - (Player->GetActorRightVector() * HalfSpacing);
OutSecondCardLoc = BaseLocation + (Player->GetActorRightVector() * HalfSpacing);
```

**문제점**:
- 플레이어가 고개를 돌리면 (카메라 회전) ForwardVector 변경
- 카드가 플레이어 시선에 따라 이상한 위치에 스폰됨

#### Community Cards
```cpp
// ❌ 문제: 항상 0도 회전 (가로 배열 고정)
FRotator CardRotation = FRotator::ZeroRotator;
```

**문제점**:
- 테이블 그림의 각도와 맞지 않음
- 회전 조정 불가능

### 성공한 방식 (After)

#### Player Cards
```cpp
// ✅ 해결: 테이블 중심 → 플레이어 위치 방향을 "앞"으로 사용
FVector ForwardDirection = (TableLocation - PlayerLocation).GetSafeNormal2D();

// Forward와 직교하는 Right 벡터 계산
FVector RightDirection = FVector::CrossProduct(FVector::UpVector, ForwardDirection);

// 카드 위치 배치
OutFirstCardLoc = BaseLocation - (RightDirection * HalfSpacing);
OutSecondCardLoc = BaseLocation + (RightDirection * HalfSpacing);

// 카드 회전
FRotator CardRotation = FRotationMatrix::MakeFromYZ(ForwardDirection, FVector::UpVector).Rotator();
```

**개선점**:
- 플레이어 시선과 무관하게 **테이블 중심을 기준**으로 방향 계산
- `GetSafeNormal2D()`로 Z축 무시, 평면상 방향만 사용
- 플레이어가 카메라를 어디로 움직여도 카드 위치 고정

#### Community Cards
```cpp
// ✅ 해결: 회전 각도 파라미터 추가 및 배열 회전
float Offset = (CardIdx - 2) * CommunityCardSpacing;
FVector LocalPos = FVector(Offset, 0, 0);

// Z축 기준으로 배열 회전
FVector RotatedPos = LocalPos.RotateAngleAxis(RotationAngle, FVector::UpVector);
FVector SpawnLocation = TableLocation + RotatedPos;

// 카드 자체도 같은 각도로 회전
FRotator CardRotation = FRotator(0.0f, RotationAngle, 0.0f);
```

**개선점**:
- `CommunityCardRotationAngle` 변수로 에디터에서 회전 각도 조정 가능
- `RotateAngleAxis`로 배열 전체를 Z축 기준 회전
- 테이블 그림에 정확히 맞출 수 있음

### 핵심 개념 정리

1. **위치(Location) vs 회전(Rotation) 분리**
   - `SpawnActor(Location, Rotation)`: Location은 회전 중심점, Rotation은 자세

2. **고정된 기준점 사용**
   - 동적 기준(플레이어 시선) → 정적 기준(테이블 중심)으로 변경

3. **평면상 회전**
   - `GetSafeNormal2D()`: Z축 무시하고 XY 평면만 고려
   - `RotateAngleAxis(Angle, UpVector)`: Z축 중심 회전
