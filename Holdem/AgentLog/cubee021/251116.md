   - `Client_OnPossess_Implementation()`: 자신이 컨트롤하는 Pawn의 NameTag만 비활성화
   - **핵심 개념 학습**: Client RPC는 해당 Actor를 소유한 클라이언트에게만 전송됨
   - 결과: 각 플레이어는 자신의 NameTag는 안 보이고, 다른 플레이어의 NameTag만 보임

3. **멀티플레이어 핵심 개념 이해**
   - **복제본(Replica) 개념**: 각 클라이언트는 월드의 모든 Actor 복제본을 가짐
   - **Tick 실행 위치**: 각 클라이언트에서 모든 Actor의 Tick이 실행됨
     - A의 PC: 모든 플레이어의 Tick 실행 → `GetPlayerCameraManager(0)` = A의 카메라
     - B의 PC: 모든 플레이어의 Tick 실행 → `GetPlayerCameraManager(0)` = B의 카메라
   - **결과**: 같은 코드가 각 클라이언트에서 다른 결과를 만듦 (로컬 카메라 참조)

4. **세션 플레이어 수 표시 시도**
   - 델리게이트 파라미터 추가 방식 논의 (`FFindComplete`)
   - `SessionWidget::SetSessionPlayersInfo()` 연동 구조 설계
   - **문제 발견**: OnlineSubsystemNull에서 `NumOpenPublicConnections`가 업데이트 안 됨
     - 세션에 플레이어가 접속해도 항상 `OpenSlots: 6` (변화 없음)
   - **원인**: NULL 서브시스템은 세션 상태를 자동 추적하지 않음
   - **해결 방향**: GameMode의 `PostLogin`/`Logout`에서 커스텀 데이터(`CURR_PLAYERS`)로 수동 관리
   - **보류**: 중요도가 낮아 다음으로 미룸
