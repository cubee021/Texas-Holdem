   - 초기 계획: 순차 번호(1, 2, 3...) 자동 배정
   - **문제점 발견**: 언리얼 세션 시스템은 분산 구조라 중앙 번호 관리 불가능
     - 동시 생성 시 중복 가능 (Race Condition)
     - 세션 검색이 비동기라서 "현재 몇 개?" 확인 중에도 다른 사람이 생성 가능
   - **최종 결정**: 플레이어 이름을 세션 이름으로 사용
     - 대안: 타임스탬프, GUID, 카지노 테이블 컨셉 등 논의
     - 순차 번호는 별도 REST API 서버 필요 (오버엔지니어링)

3. **MainMenuWidget 크래시 해결**
   - 증상: `EXCEPTION_ACCESS_VIOLATION` (nullptr 접근)
   - 원인: `GI` (GameInstance) 변수가 초기화되지 않음
   - 해결: NativeConstruct에서 `GI = Cast<UHoldemGameInstance>(GetGameInstance());` 추가

4. **OpenLevel vs 세션 생성 차이 이해**
   - **OpenLevel**: 단순히 Listen Server 레벨만 열기, 세션 목록에 안 뜸, IP 직접 입력 필요
   - **세션 생성**: Online Subsystem에 세션 등록 → 다른 플레이어가 검색 가능
   - 현재 구현: 세션 생성 성공 시 `ServerTravel`로 레벨 이동 (올바른 방식)
