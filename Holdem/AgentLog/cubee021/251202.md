- Folded (게임 참여 상태): Active, Folded, AllIn
- Dealer/Blind (포지션/역할): None, Dealer, SmallBlind, BigBlind
- 핵심 발견: **한 플레이어가 "Dealer이면서 Folded" 가능** → 다른 차원의 정보!

**결정: 상태와 포지션 분리**
- `bIsFolded` (bool): 폴드 여부
- `Position` (EPlayerPosition enum): 포지션
- 이유:
  - Folded는 "액션"이라 명시적 저장 필요
  - Position은 매 라운드 바뀌는 역할
  - 두 정보는 독립적이고 조합 가능

**AllIn 처리 방식 논의**
- 별도 상태로 저장 (bIsAllIn) vs 칩 개수로 판단
- 질문: "AllIn은 상태로 따로 분리할 필요가 있을까?"

**AllIn 특징 분석**
- AllIn 플레이어도 승자 판정 포함 (Showdown 참여)
- 베팅 라운드에서 액션 불가 (칩이 없어서)
- 게임 포기 아님 (계속 참여 중)
- **핵심**: AllIn은 "상태"가 아니라 칩 개수의 "결과"

**결정: AllIn은 계산으로 판단**
- `CurrentChips == 0 && !bIsFolded` → AllIn
- `IsAllIn()` 헬퍼 함수로 제공
- 이유:
  - 중복 저장 문제 방지 (CurrentChips + bIsAllIn)
  - Single Source of Truth 원칙
  - 동기화 버그 방지
